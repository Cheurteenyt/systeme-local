# Security Policy

Signalez les vulnérabilités de façon privée. Ne publiez jamais de preuve de concept contenant des secrets, une méthode d'évasion exploitable sur des machines tierces ou un mécanisme de persistance furtif.

Le projet applique les règles suivantes :

- pas de shell hôte exposé ;
- pas de contrôle distant sans consentement explicite ;
- pas de contournement des restrictions d'un fournisseur ;
- pas de secret dans les prompts ou journaux ;
- toute nouvelle capacité doit inclure tests, politique et documentation de menace.
